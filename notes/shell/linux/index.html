Basic command

Clear the screen

clear

problem

When terminals database is inaccessible happened.

Add xport TERMINFO=/usr/share/terminfo to .bashrc

Permissions

Give read, write, and execute permissions for everyone

chmod -R 777 <dir>/<folder_name>

Copy

cp <dir>/file_name> <dir>/<new_name>
cp -r <dir>/* <dir>

Move

mv <dir>/<file_name> <dir>/<new_name>

It also could be used to change the name of file when two <dir> are same.

Pathway

pwd

Folder

Create folder

mkdir <dir>/<name>

e.g. mkdir /home/user/new_folder

Check content of folder

ls
ls <dir>

Files count

ls -l | grep "^-" | wc -l (Count files number in this path, NOT containing subpath)
ls -lR| grep "^-" | wc -l (Count files number in this path, containing subpath)
ls -lR | grep "^d" | wc -l (Count folders number in this path, containing subpath)

Search file

locate <name>
find <name>

kill

kill <pid>
kill -9 <pid> (-9 stands for mandatory)
killall <program_name>
pkill <program_name>
xkill (Available on graphical interface)

touch

touch <file_name>

Create a new file(Do nothing if the file already exists)

Apt

Please be kindly notified that apt-get could be short to apt when using

Install new package

sudo apt-get install <package_name>

Update dict

sudo apt-get update

Update installed package

sudo apt-get upgrade
sudo apt-get upgrade --only-upgrade <package_name>

NVIDIA GPU

CUDA version

nvcc -V
nvcc --version

GPU status

nvidia-smi
watch -n 1 -d nvidia-smi
gpustat (install it with conda/pip install gpustat)
gpustat -i
nvtop

GPU driver

ubuntu-drivers devices
glxinfo | grep rendering
sudo apt search nvidia-driver | grep nvidia-driver

System info

CPU

Whole info

cat /proc/cpuinfo

CPU model

cat /proc/cpuinfo | grep name | cut -f2 -d: | uniq -c

Core number

cat /proc/cpuinfo | grep physical | uniq -c

Bit width (32/64)

getconf LONG_BIT

Memory

Whole info

cat /proc/meminfo

Short info

free -h

Dynamic info

top or htop

Process status

ps <option>

<option> could be one of below:

-A (List all process)
-w (Wider list)
-au (List with more details)
-aux (List all processes of all users)
-ef (List of all process with terminal type info)
-u root (List the processes run by root)
-ef | grep <key_word> (List of processes with given key word)

Storage

Whole info

df -h

Info of current device

df -h <dir>

e.g. df -h ./

Allocated space

du -sh <dir>

All folder’s allocated space

du -sh <dir>/*

Network interface controller/card

dmesg | grep -i eth

Kernel

uname -a

Ubuntu version

lsb_release -a

Offline program

screen

Installation

apt install screen

Create a new screen

screen -S <name>

Check all screen

screen -ls

Make screen offline

screen -d <name>

Recover screen

screen -r <name>

Back to terminal

ctrl+A+D

Delete screen

screen -X -S <name> quit
screen kill <name>
screen kill <id>

Downloads & Upload

wget

wget [option] [url]

[option] could be blanck

Change file name

wget -O <new_name> <url>

Change file path

wget -P <new_path> <url>

Continute downloading after break

wget -c <url>

Increase attempts

wget -t <num> <url>

The default <num> is 40

use ftp

wget --ftp-user=<username> --ftp-password=<password> <url>

Simulate browser

wget -U '<browser_name>' <url>
<browser> could be one of {Mozilla/5.0 (Windows NT 10.0; Win64; x64), AppleWebKit/537.36 (KHTML, like Gecko), Chrome/81.0.4044.43, Safari/537.36}

Downloads on background

wget -b <url>

Check download log

tail -f wget-log

multi-files download

vim downloads.txt
wget -i downloads.txt

curl

This command could also be used on Windows/MacOS

curl -o <name> <url>

Continute downloading after break

curl -C - -o <name> <url>

Increase attempts

curl --retry 3 -o <name> <url>

scp

Download files from server to local

scp username@servername:/remote_path/filename ~/local_destination

Upload files from local to server

scp ~/local_path/local_filename username@servername:/remote_path

Download whole directory from server to local

scp -r username@servername:/remote_path/remote_dir/ ~/local_destination

Upload whole directory from local to server

scp -r ~/local_dir username@servername:/remote_path/remote_dir

Use scp -P PortId ... instead if you need to specify a port to connect to the server

Execution

.bin

Using chmod command to assign the execution permission to the file with ‘.bin’ suffix.

chmod +x filename.bin

And then, the file with ‘.bin’ suffix could be executed by the following.

./filename.bin

nano

Usage

nano ./file.sh

Jump to the end of file

Alt+/

Jump to given word

Ctrl+w

Input the word you want to find and it will help you.

Ctrl+v

This could also help you reach the end of file

.sh

./file.sh
bash ./file.sh
source ./file.sh

Construct a .sh file

cd <path>
conda activate <env_name>
python main.py --data 1 &
python main.py --data 1 &&
python main.py --data 1 ||

In file.sh like above:
& stands for executing simultaneously the command of this line and next line.
&& stands for executing the command of this line and next line one by one.
|| stands for terminating the execution of next command when command of this line successfully executed.
