Usage

Check installed package

conda list

Check virtual environment

conda env list or conda info -e

Update conda

conda update conda

Create env

conda create -n <env_name> python=3.x

Remove env

conda remove -n <env_name> --all

Activate/Deactivate env

For Linux/MacOS

source activate <env_name>

source deactivate <env_name>

For Windows/Linux/MacOS

conda activate <env_name>

conda deactivate <env_name>

Install package

conda install <package_name>=<version>

e.g. conda install numpy=1.22.0

Remove package

conda uninstall <package_name>

Clone env

conda create -n <new_env> --clone <old_env>

Clean conda

conda clean --all

Remove package

conda remove -n <env_name> <package_name>

Usage of environment.yaml

Export env into yaml file

conda env export > environment.yaml

Create new env from yaml file

conda env create -f environment.yaml

Usage of requirements.txt

conda install --yes --file requirements.txt

Debug examples

NoWritableEnvsDirError

NoWritableEnvsDirError: No writeable envs directories configured.
  - /home/<username>/.conda/envs
  - /home/<username>/anaconda3/envs

Find the root path of ‘.conda’ file and use ‘cd’ enter that path and run the below commmand.

sudo chmod a+w .conda

or enter the path of ‘anaconda’ folder and run the below command.

sudo chown -R <username> anaconda
